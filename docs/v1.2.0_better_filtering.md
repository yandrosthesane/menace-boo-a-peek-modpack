# v1.2.0 — Dynamic Faction Discovery

Removes all hardcoded faction indices from BooAPeek. The mod now discovers factions at runtime and handles any faction layout, any number of player units, and allied AI factions correctly.

## Problem (v1.1.1)

Four hardcoded constants assumed a fixed faction layout:

```csharp
private const int FACTION_PLAYER_1 = 1;
private const int FACTION_PLAYER_2 = 2;
private const int FACTION_AI_MIN   = 3;
private const int FACTION_AI_MAX   = 9;
```

This caused several issues:

| Issue | Impact |
|-------|--------|
| Only factions 1-2 treated as player | Breaks if mods add player factions or the game changes indices |
| Only factions 3-9 treated as AI | Misses any modded factions beyond index 9 |
| `GetAIFactionProxy(6)` hardcoded to Pirates | Reflection init fails if Pirates don't exist in the mission |
| No allied faction check | AlliedLocalForces (faction 4) had opponents stripped even when allied with the player, potentially degrading allied AI behavior |

## Solution — Runtime Faction Probing

### Discovery Algorithm (runs once per tactical scene, after 60-frame init delay)

```
For each faction index 0..15:
  1. Call TacticalManager.GetFaction(i)
     → null? Skip (faction doesn't exist in this mission)

  2. Type-check via GameObj.Is(aiFactionGameType)
     → false? It's a BaseFaction (player-side). If it has living actors, add to _playerFactionIds.

  3. Confirmed AIFaction — construct managed proxy

  4. Read AIFaction.m_IsAlliedWithPlayer
     → true?  Add to _alliedAiFactionIds (filtering SKIPPED)
     → false? Add to _aiFactionIds (filtering ACTIVE)

  5. Use first available opponent list to discover List<Opponent> generic type
```

### Three-Way Classification

```
_aiFactionIds          HashSet<int>   Hostile AI — opponent filtering ACTIVE
_alliedAiFactionIds    HashSet<int>   Allied AI  — opponent filtering SKIPPED
_playerFactionIds      List<int>      Non-AI factions with living actors
```

### Turn Detection

Before (range check):
```csharp
if (currentFaction >= FACTION_AI_MIN && currentFaction <= FACTION_AI_MAX)
```

After (set lookup):
```csharp
if (_aiFactionIds.Contains(currentFaction))
```

### Player Unit Enumeration

Before (hardcoded to 2 factions):
```csharp
var actors1 = EntitySpawner.ListEntities(FACTION_PLAYER_1);
var actors2 = EntitySpawner.ListEntities(FACTION_PLAYER_2);
```

After (iterates all discovered player factions):
```csharp
foreach (int factionId in _playerFactionIds)
{
    var actors = EntitySpawner.ListEntities(factionId);
    ...
}
```

## Why `GameObj.Is()` for Type-Checking

Il2Cpp proxy constructors (`new AIFaction(IntPtr)`) don't validate the underlying type — they just store the pointer. Calling `GetAIFactionProxy(i)` on a non-AIFaction (e.g., Player faction) would return a non-null proxy that crashes when you invoke AIFaction methods on it.

The fix: before constructing the proxy, check the actual Il2Cpp class type:

```csharp
var factionGameObj = new GameObj(((Il2CppObjectBase)factionObj).Pointer);
if (!factionGameObj.Is(aiFactionGameType))
    continue;  // Not an AIFaction, safe to skip
```

## Why Skip Allied Factions

`AIFaction.m_IsAlliedWithPlayer` marks factions that fight alongside the player (e.g., AlliedLocalForces in certain missions). Their `m_Opponents` list contains hostile enemies, not player units. Stripping unseen opponents from an allied faction would make them blind to enemies they should be fighting.

## FACTION_PROBE_MAX = 15

The known `FactionType` enum goes 0-9. The probe upper bound of 15 gives headroom for modded factions without probing indefinitely. If the game adds more factions, bump this single constant. Out-of-range indices return null from `GetFaction()` and are silently skipped.
